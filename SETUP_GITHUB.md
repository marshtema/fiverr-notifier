# 🚀 Запуск на GitHub Actions (бесплатно, 24/7, без карты)

Пошаговая инструкция. Делается один раз.

## Шаг 1. Создать репозиторий

1. Зайди на https://github.com → кнопка **New** (или https://github.com/new).
2. Repository name: `fiverr-notifier` (или любое).
3. Выбери **Public** (тогда минуты Actions бесплатны без лимита; секреты всё равно
   спрятаны в Secrets, в коде их нет) — или **Private**, но тогда интервал лучше
   сделать пореже (см. ниже).
4. Нажми **Create repository**.

## Шаг 2. Залить код

Самый простой способ — через сайт:
1. На странице репозитория → **uploading an existing file**.
2. Перетащи туда все файлы из папки `fiverr-notifier`
   **КРОМЕ `.env`** (его заливать нельзя — там секреты!).
3. Убедись, что папка `.github/workflows/notifier.yml` загрузилась.
4. Commit.

> Файл `.env` защищён через `.gitignore`, но при ручной загрузке через сайт
> просто не добавляй его сам.

## Шаг 3. Прописать секреты

В репозитории: **Settings → Secrets and variables → Actions → New repository secret**.
Добавь по одному (имя → значение):

| Name | Value |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен бота |
| `TELEGRAM_CHAT_ID` | твой chat_id |
| `GMAIL_ADDRESS` | адрес Gmail |
| `GMAIL_APP_PASSWORD` | App Password Gmail |
| `GEMINI_API_KEY` | ключ Gemini |

## Шаг 4. Включить и проверить

1. Вкладка **Actions** → если просит — нажми «I understand, enable workflows».
2. Слева выбери **Fiverr Notifier** → справа **Run workflow** (ручной запуск для теста).
3. Через минуту проверь Telegram и зелёную галочку в Actions.

Дальше робот будет сам запускаться каждые 10 минут. Готово! 🎉

## Интервал и лимиты

- Файл расписания: `.github/workflows/notifier.yml`, строка `cron: "*/10 * * * *"`.
- **Public репозиторий** — минуты бесплатны без ограничений, можно хоть `*/5`.
- **Private репозиторий** — 2000 мин/мес бесплатно. Чтобы влезть, держи интервал
  `*/15` или реже (`*/10` ≈ 4300 мин — превысит лимит на приватном).
- Поменять интервал: отредактируй строку `cron` прямо на GitHub и сделай commit.

## Если что-то не пришло

- Actions → открой последний запуск → посмотри лог шага
  «Проверить почту и отправить уведомления».
- Частые причины: опечатка в Secret, или у бота не нажат Start.
